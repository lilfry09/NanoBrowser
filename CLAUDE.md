# 自动开发 Agent 指南

## 项目上下文
这是 `D:\SJTUlearning\25Winter` 的工作目录。我们将使用 AI 自动开发系统在此进行开发。具体开发需求请阅读 `task.json`。

## 工作流 (MANDATORY)
每一个 AI 会话必须严格遵循以下流程：

### Step 1: 接单
读取 `task.json`，找到并选择第一个状态为 `passes: false` 的任务。

### Step 2: 实现功能
仔细阅读任务的 `description` 和 `steps`。
根据需求使用 Bash, Write, Edit, Read 等工具，编写或修改代码来实现该功能。
**尽量保持代码风格一致，如果有现有代码，请参考现有代码。**

### Step 3: 测试验证
在提交代码之前，请**务必**执行相关的测试或验证命令（如编译、lint、运行单元测试），确保你写的代码是可用的、无明显语法或逻辑错误的。

### Step 4: 更新进度记录
在 `progress.txt` 的底部追加你的工作日志。
记录格式如下：
```markdown
## [日期时间] - 完成任务: [任务标题]
- 执行了哪些操作（创建了什么文件，修改了什么逻辑）
- 测试方法与结果
- 给人类或未来 Agent 的注意事项
```

### Step 5: 更新 task.json 并 Commit (强制要求)
1. 使用工具更新 `task.json`，将该任务的 `"passes": false` 修改为 `true`。**不要删除原有任务，只改状态！**
2. 执行以下命令，将当前任务所有的变更（包括代码、`task.json` 和 `progress.txt`）一次性提交：
```bash
git add .
git commit -m "完成任务: [任务标题]"
```

---

## ⚠️ 遇到阻塞 (Blocking Issues)
如果你遇到了**完全无法通过代码解决**的障碍（例如：必须人类提供密码/API Key、必须人类扫码登录、服务器彻底宕机连不上等），**千万不要**假装完成任务！

遇到此类阻塞时：
1. **不要**提交 Git Commit。
2. **不要**修改 `task.json` 的 passes 为 true。
3. 在 `progress.txt` 底部写明阻塞原因，以及你需要人类帮你做什么。
4. 退出当前任务。人类看到日志后会介入处理。
